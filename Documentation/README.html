Avatar License Manager
by StarSideUp
Unity 2022.3+ VRChat Avatar SDK Editor Only v0.1.0
アバターの Prefab を Unity Editor にドロップするだけで、そのアバターが参照している すべてのアセット（衣装・ヘア・シェーダーなど）を自動解析。製品ごとにライセンスファイルの 場所をひとめで把握できます。

Drop your avatar Prefab into the tool — it scans all asset dependencies, groups them by product, and shows you where each license file lives.

製品の自動グループ化について
Unity の AssetDatabase が管理するすべての依存アセットのパスを解析し、 Assets/<Vendor>/<Product> の階層パターンを製品ルートとして自動認識します。
例: Assets/SomeShop/ExampleOutfit/Prefabs/Outfit.prefab → 製品ルート: Assets/SomeShop/ExampleOutfit（販売者: SomeShop、製品名: ExampleOutfit）

自動抽出した販売者名・製品名はウィンドウ上で手動編集できます。
✦

主な機能 / Features

Prefab ドロップで依存解析
アバター Prefab をドロップするだけで、参照しているすべてのアセットを収集し 製品単位に自動グループ化します。Project View でも Hierarchy でも対応します。
ライセンスファイルの自動検出
各製品フォルダ内を再帰スキャンし、利用規約.txt や avatar-license.json などを自動検出してパスを一覧表示します。
Ping / 開く
検出したライセンスファイルを Project View にハイライト（Ping）するか、 OS の既定アプリ（PDF ビューワーなど）で直接開けます。
製品ごとのメモ
製品ごとに改変可否・クレジット要否などのメモを自由に記入できます。 分析結果とともに ScriptableObject として保存されます。
販売者名・製品名を編集可能
自動検出した名前が正しくない場合は、ウィンドウ上で直接編集できます。 編集内容は次回分析時にも引き継がれます。
セッション間で保持
分析結果とメモは ScriptableObject として保存され、Unity を再起動しても 前回の状態が復元されます。

使い方 / How to Use

1
ウィンドウを開く
Unity メニューの Tools > StarSideUp > Avatar License Manager を選択します。
2
アバター Prefab を指定する
以下のいずれかの方法でアバターを指定します。
① ドロップ — Project View または Hierarchy からアバター Prefab を ウィンドウ上部のドロップゾーンにドラッグ＆ドロップします。
② ObjectField — ドロップゾーン下の「Avatar Prefab」欄に直接ドラッグします。
③ 選択中を使う — Project View または Hierarchy でアバターを選択してから 「選択中を使う」ボタンを押します。
3
「分析する」を押す
「分析する / Analyze」ボタンを押すと、Prefab の推移的依存アセットを解析し、 製品ルートを自動検出してライセンスファイルをスキャンします。
4
製品カードでライセンスファイルを確認する
検出された製品が一覧表示されます。各カードに含まれる情報:
・ライセンスファイルのパス（Assets/... 形式）
・Ping → Project View でファイルをハイライト
・開く → OS 既定アプリで開く（PDF ビューワーなど）
ライセンスファイルが見つからない場合は警告アイコンが表示されます。
5
名前を修正する（必要に応じて）
自動検出した販売者名・製品名がフォルダ名と異なる場合は、カード内の 「販売者」「製品名」欄を直接編集できます。
6
メモを記入する（任意）
各製品カードのメモ欄に、利用規約の要約・改変可否・クレジット要否などを 自由に書き残せます。再分析時にもメモは引き継がれます。

ライセンスファイルの検出ルール / Detection Rules

製品ルートフォルダ（Assets/<Vendor>/<Product>）を起点に、 以下の優先順でファイルを検索します。

優先①：機械可読 JSON（製品ルート直下）
avatar-license.json license.json vn3-license.json
優先②：ライセンスフォルダ内のすべての文書ファイル
フォルダ名が以下のいずれかと一致する場合、中の文書ファイルをすべて対象にします。
LicenseLicensesVN3 Terms規約利用規約ライセンス
優先③：ファイル名にキーワードを含む文書ファイル（再帰スキャン）
検索キーワード対象拡張子
license, licence, ライセンス, 利用規約, terms, eula, vn3 .txt  .pdf  .md  .html  .htm
⚠
検出されない場合： ライセンスファイルのファイル名・フォルダ名が上記パターンに合わない場合は未検出になります。 その場合は製品カードの「Ping」でフォルダを Project View に開き、手動でファイルを確認してください。 再分析すると最新の状態で再スキャンされます。

注意事項 / Notes

法的助言ではありません
本ツールはライセンスファイルの所在を把握するための補助ツールです。 利用可否の最終判断は利用規約の原文をご自身で確認のうえ行ってください。 本ツールの出力は法的根拠を持ちません。
アセット本体を変更しません
依存解析と表示のみを行います。Prefab・マテリアル・アニメーターなど、 分析対象のアセットファイルを変更することは一切ありません。
製品グループ化の前提
Assets/<Vendor>/<Product> という 3 階層のフォルダ規則に 基づいて製品を認識します。この規則に従っていないアセットは別の製品として グループ化されるか、検出されないことがあります。販売者名・製品名は手動修正できます。
Editor 専用ツール
Unity Editor でのみ動作します。ビルドには含まれず、 VRChat のアバター・ワールドのパフォーマンスには一切影響しません。
